# 支出カテゴリーマスター機能 - 大枠設計

## 概要
予算管理の支出カテゴリー作成・管理機能。ユーザーが「スマホ代」「食費」などのカテゴリーを作成し、支出分類に使用する。

## バックエンド設計

### DynamoDB テーブル
**テーブル名:** ExpenseCategoryMaster
- PK: `user_id` (STRING)
- SK: `category_id` (STRING, UUID)
- 属性: `category_name`, `color` (HEX), `icon_emoji`, `is_deleted`, `created_at`, `updated_at`

### API エンドポイント
```
POST   /expense-categories          - 作成（name, color, icon_emoji）
GET    /expense-categories          - 一覧（削除済み除外）
PUT    /expense-categories/{id}     - 更新
DELETE /expense-categories/{id}     - 削除（論理削除）
```

## フロントエンド設計

### ページ構成
- **ルート:** `/budget/categories`
- **コンポーネント:**
  - `ExpenseCategoryList.svelte` - テーブル表示
  - `ExpenseCategoryForm.svelte` - 作成/編集フォーム

### 機能
- カテゴリー一覧表示（カラム: 名前・色・アイコン）
- 新規作成ボタン → モーダルフォーム
- 編集・削除アクション
- リアルタイム反映（レスポンス後 UI 更新）

## 技術スタック
- Backend: FastAPI + Pydantic + aioboto3
- Frontend: SvelteKit + Svelte 5 Runes
- API: OpenAPI SDK（自動生成）
