# ワークフロー: [ドキュメント化]

## フロー

```
① コードベース把握（gemini-code-overview）
   - docs/overview-*.md キャッシュを先に確認
   - キャッシュが古い・存在しない場合のみ再実行

② ドキュメント生成（gemini-docs）
   - 出力先: docs/{名前}.md（interface/ には出力しない）

③ ユーザーへ報告 → /clear
```

## overview キャッシュの再実行タイミング

- キャッシュ行数 > 300
- ファイルの追加・削除あり
- 複数ファイル横断の大規模変更あり

## 完了
